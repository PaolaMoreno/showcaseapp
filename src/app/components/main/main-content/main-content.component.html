DPort web app

A digital portfolio with mailing and shcheduling capabilites enables business consultants to seal deals with health institutions. One of my task was designing and implementing the Layout. Here I will describe some of the concepts used to create the design.


The picture portraits the authoring view, a web app feature that enables the user with administrator credentials to edit the portfolio layout. That is, the libraries background color, name, position, sublibraries, documents, texts and Internet links.

The preview container shows instant changes that can be publish upon user request.

This design is inspired mainly by Gestalt Principles and Heuristics, two sets of principles that emerge from the study on Human Factors and its application to the Human Computer Interaction. On the technical axes, it is developed with Angular 6, HTML5, CSS3 and the Angular Material library.

For reasons of confidentiality, a minified and abstract Mock-up version of the application is displayed here. Libraries are meant to signify grouping categories.

Gestalt principles
Researched by German psychologists...Did you see that? Yes, Germans! ;) These principles attempt to describe how people perceive visual elements when certain conditions apply.
  • Proximity
    The image shows in two ocassions an element named "Library a", but to recognise that they have different functionalities attached, they are placed further apart.
  • Common Region
    However, placing them further apart would not necessarily fulfill the purpose of conveying different functionality. To do that, the first library is placed in the container named "Preview" and the second in the lower container that hosts also an "Add" call to action button. This button is placed there so that a Library can be added to the lower container but not to the first one. The same happens with the container where "Documents texts and links" are displayed (soft purple container). Rather than being pure libraries (grouping categories if you like) they host content, like pdfs or excel documents and that is the reason why they are enclosed in a different container, whithin its parent container, the library.

Heuristics
  • Visibility of system status
    The application is in charge of informing appropriately changes and states of the system. In the lower container there are a set of tooltips that would be displayed upon user hovering, they tell the user why to click on them and what to expect after that. Another clear example is shown in the navigation bar. To know where in the navigation a user currently is, the active tab must be highlighted.
    A similar pattern is shown with the breadcrumbs that enablee users to go back from one sublibrary to its library parent.
  • Recognition rather than recall
    This principle advocates to help users minimize their memory load by making objects, actions, and options visible. To this purpose, standard icons with a label (when possible) are displayed as opposed to creating "unique" icons whithout labels or tooltips that leave our users wondering how to achieve a goal with our user interface or even worse deleting files that were not allowed to be deleted, for example.

DPort iPad app

In the iPad version of the application, no authoring view is enabled to favoured device capabilities. A minified version of the main screen is shown.

Responsiveness
To support responsiveness, the grid that in bigger viewports outlines three or more columns, is now displayed in one single column containing all libraries.

Accessibility
according to the W3C Accessibility guidelines, level AA requires a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text. Level AAA requires a contrast ratio of at least 7:1 for normal text and 4.5:1 for large text.
Because of this, the font color of some libraries is dark and for the others it is bright.
The font sizes are fixed in rem units to create a perfectly scalable layout.
