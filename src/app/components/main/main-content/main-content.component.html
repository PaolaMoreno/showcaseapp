DPort web and iPad app

A digital portfolio with mailing and scheduling capabilites enables business consultants to seal deals with health institutions. The application consists of a series of grouping categories with consulting or campaign documents, contracts, links and descriptions that are displayed in an iPad and used for consultants to attend business meetings.
The categories, which for this purpose are named "libraries", where showcased in the landingpage inside a grid system with well defined colors and positions. I was then in charge of designing and implementing the layout editor. Here I will describe some of the concepts the design was based on.


For reasons of confidentiality, a minified and abstract Mock-up version of the feature is displayed here.

The picture displays the authoring view, a web app feature that enables the user with administrative credentials to edit the portfolio layout. That is, the libraries background color, name, position, sublibraries, documents, texts and links.

The preview container shows instant changes that can be published upon user request.

This design was mainly inspired by Gestalt Principles and Heuristics, two sets of principles that consider human factors and are applied to the interaction with the computer. On the technical side of things, it was developed with Angular 6, HTML5, CSS3 and the Angular Material library.

Gestalt principles

Developed by German psychologists...Did you see that? Yes, Germans! ;) These principles attempt to describe how people perceive visual elements when certain conditions apply.

  • Proximity

    The picture at the left shows in two ocasions an element named "Library a", but to recognise that they have different functionalities attached, they are placed further apart.

  • Common Region

    However, placing them further apart would not necessarily fulfill the purpose of conveying different functionality. To do that, the first library is placed in the container named "Preview" and the second in the lower container that hosts also an call to action button. This button is placed there so that a Library can be added to the lower container but not to the first one. The same happens with the container where "Documents texts and links" are displayed (soft purple container). Rather than being pure grouping categories they host content, like pdfs or excel documents and that is the reason why they are enclosed in a different container, within its parent container, the library.

Heuristics
  • Visibility of system status

    The application should appropriately inform the user when changes or different states of the system occur. In the lower container there are a set of tooltips that would be displayed upon user hovering, they tell the user why to click on them and what to expect after that. Another clear example is shown in the navigation bar. To know where in the navigation a user currently is, the active tab must be highlighted.
    A similar pattern is shown with the breadcrumbs that enable users to go back from one sub library to its library parent.

  • Recognition rather than recall

    This principle advocates to help users minimize their memory load by making objects, actions, and options visible. To this purpose, standard icons with a label or tooltip are displayed across the application, as opposed to creating "unique" icons that could potentially leave our users wondering how to achieve a task or even worse, deleting files without permission.

DPort iPad app

In the iPad version of the application, no authoring view is enabled to favoured device capabilities. At the top, the libraries are displayed and the sublibraries appear upon user taps.
The main design concepts, emerge here from responsive design, accessibility and again heuristics.

Responsiveness

To support responsiveness, the grid that in bigger viewports outlines three or more columns, is now displayed in one single column containing all libraries.

Accessibility

According to the W3C Accessibility guidelines, level AA requires a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text. Level AAA requires a contrast ratio of at least 7:1 for normal text and 4.5:1 for large text. This is the reason why the font color of some libraries' names is dark or bright, depending on its background color.
The font sizes are fixed in rem units to create a perfectly scalable layout.

Color contrast is a must check to ensure readability and an outstanding user experience.

Heuristics

Following the "visibility of system status" heuristic, the icon "send" has been given a different cue to signify whether it is or not in the list of attachments that will later, upon user request, be sent via Email. The light gray stands out for a documents that have not been added to the list and the black for a document that actually is in the list of attachments, waiting to be sent.

On the upper right corner, the "Email" icon displays the number of documents in the list and is updated as soon as the user taps on the Email icon in the document container. Given that the list of documents may be bigger than the device's available height, a search function is included to ease the selection process. In case that the list surpases by a big amount that height, table pagination and sorting capabilities are included.

UX Designer

Following the introduction of new touch based ATMs (cash machines) at NCR in Edinburgh, services built with web technologies would afterwards be deployed to the cash machine via ATM network. New services and more interactions that had ever been possible could be allowed, as the device was not restricted anymore to the traditional "8 hardware buttons to the side". This brought four interesting research questions that were the focus of a research study that culminated with a web prototype.

1) ¿Would users adapt to the new services and interactions?

2) ¿Would the new technology make transactions faster?

3) ¿How would different age groups respond to this novelty?

4) ¿How could discoverability of new services be achieved?


User Research

In order to take in the third question, two age groups were invited to participate in the study. Under 30 and over 60 years old. I then carried out the first collaborative design session. A set of four "main screen" plus "additional services screen" were presented to the audience and via grounded theory, two designs were selected for each group.
The results exposed here will refer only to the younger group.

Adaptability to new services

To investigate how users would think of the introduction of new services like buying events' tickets, I conducted a survey and 57 participants took part in.
Based on the results, the 3 most popular additional services were displayed on the main screen next to an invitation to discover more.

Discoverability

Color can certainly influence the attractiveness of an element. I did then created a color palette for the interactive mockups and and carried out a second study with a different group of young participants to reinforce external validity. Color was not enough to know if the design would boost discoverability. I then introduced eye tracking technology to study the fixation time on the "invitation to discover" area, as this is related to repeated attention, and this in turn also a measurement of engagement according to research done on user modelling for web applications.


Transaction time

A second study was also conducted to inquire whether the technology could reduce transaction time and measure Usability. For that purpose, using counterbalancing to avoid learning effects, a withdrawal of 20 and 50 pounds was given as a task to the users.

What were the results achieved?

The main challenge that aroused after getting the survey results were that participants had barely conceived of cash machines as a providers of additional services. The traditional user would approach the machine to get money, eventually check balances and leave as soon as possible.
This may be the same case in Germany?

  • Adaptability

    This attitude makes adaptability a characteristic difficult to achieve. Therefore, the introduction of new technology that eventually modifies its functionality should initially the less context-disruptive possible. In this sense, this user interface displays the traditional services, plus two or three more that are banking related.

  • Transaction time

    Provided the ATM network does not introduce latency, looking at the results of this study it can be said that touch technology and a specific design makes transactions faster than the market average of 45 seconds.

  • Age groups

    Adaptability is linked to personal characteristics, and although the attitude towards adaptability seemed to be the same across age groups, this trait got more accentuated among the seniors. This is linked to the rather uncomfortable idea of communicating with a machine as opposed to with a person at the ticketing office, for example; and to the conceptual model associated to the cash machine, which in some cases raised concerns on security.

  • Discoverability

    Research done by Jacob Nielsen on fixations and time spent on websites, revealed that people spent most of the time looking at information that is displayed above the page fold, against a 20% below it. In a similar study he found that content reading followed a F-shaped pattern, two horizontal stripes followed by a vertical stripe. The results of the eye tracking session showed that indeed the top left corner obtained the higher score in repeated interest, followed by the centre of the screen. For that reason, the invitation to discover additional services was placed on this area with a slight animation brought up from attention theories. One of them says that muted colour elements placed in the periphery of people’s visual field are often not noticed, in order to get them noticeable, such elements should move, so that the fovea is guided towards it. Using animate.js a subtle animation was included to draw attention to the additional services.
